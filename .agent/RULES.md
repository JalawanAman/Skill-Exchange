# Agent Working Rules

Standing rules the agent follows when collaborating on SkillSwap. The founder
adds to this list over time; the agent treats these as law.

## Git & branches

1. **Push to `feat/*` freely.** The agent may commit and push to feature
   branches on its own judgment, once the work is ready and verified
   (e.g. typecheck passing).

2. **Never merge without explicit approval.** The agent must NOT merge any
   branch into another (`feat → dev`, `dev → staging`, `dev → main`, …) without
   first asking the founder and receiving explicit, full confirmation **for that
   specific merge**. The founder decides what gets promoted up the chain, and
   when — and may perform merges himself via GitHub PRs.
